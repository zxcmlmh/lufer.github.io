Linux下安装V2rayA以使用Trojan代理
2021-03-03|Linux
字数总计: 601|阅读时长: 2 分钟

  最近推行国产化，在用UOS，其前身是Deepin，但是日常访问github时好时坏，所以还得折腾一下代理。

V2rayA

  https://github.com/v2rayA/v2rayA

  v2rayA 是一个支持全局透明代理的 V2Ray Linux 客户端，同时兼容SS、SSR、Trojan、PingTunnel协议。
  v2rayA 致力于提供最简单的操作，满足绝大部分需求。
  得益于Web客户端的优势，你不仅可以将其用于本地计算机，还可以轻松地将它部署在路由器或NAS上。

  v2rayA提供了三种使用方法：

  • 软件源安装
  • docker
  • 二进制文件、安装包

  我尝试了第一种，但是在我添加apt库的时候报错了，网上也没搜到解决方案。

  所以我干脆直接就用第二种方法装了。

ProxyChains

  由于很多程序和服务的下载都需要通过 npm, gem, nvm, git等命令进行，而在国内下载速度较差，所以还得为终端的命令启用代理，这里我采用了ProxyChains。

  https://github.com/haad/proxychains/

安装配置

  然后在最后的ProxyList里加入本地的代理设置，例如V2rayA的默认端口是20170。

  然后可以测试一下代理是否成功。
  终端运行curl -4 ip.sb，将显示自己的IP。
  终端运行proxychains curl -4 ip.sb，将显示代理服务器的IP。
  后续使用的命令行需要代理时，只需要在前面加上proxychains即可。如proxychains npm install。
